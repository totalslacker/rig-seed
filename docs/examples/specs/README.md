# Example SPECS.md Files

These are starter specs you can copy into your project's `SPECS.md` after
forking rig-seed. Pick the one closest to what you're building and customize it.

## Available Examples

| File | Project Type | Description |
|------|-------------|-------------|
| `cli-tool.md` | CLI application | A command-line tool with subcommands and config |
| `web-api.md` | HTTP API service | A REST/JSON API with database and auth |
| `library.md` | Reusable library | A library package published to a registry |

## How to Use

1. Pick the example closest to your project
2. Copy it to `SPECS.md` in your project root
3. Replace the placeholder sections with your actual requirements
4. Delete sections that don't apply

The evolution agent reads `SPECS.md` on every session to guide its work.
Be specific — vague specs produce vague results.

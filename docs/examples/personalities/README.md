# Personality Variants

Example PERSONALITY.md files for different project voices.

## Variants

| File | Style | Best for |
|------|-------|----------|
| [formal.md](formal.md) | Professional, precise | Enterprise tools, compliance-heavy projects |
| [casual.md](casual.md) | Conversational, honest | Personal projects, dev tools, open-source |
| [minimal.md](minimal.md) | Terse, output-focused | Infrastructure, internal tools, CLI utilities |

## Usage

1. Pick the variant closest to your project's voice
2. Copy it to `PERSONALITY.md` at the repo root
3. Edit to fit — these are starting points, not prescriptions

The evolution agent reads PERSONALITY.md at the start of each session to
calibrate its communication style. The more specific your examples, the
more consistent the voice.
